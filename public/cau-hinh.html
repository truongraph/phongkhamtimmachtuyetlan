Cấu hình hệ thống

Tài liệu kỹ thuật: kiến trúc, Supabase, biến môi trường, đa phòng khám, bảo mật RLS, triển khai và xử lý sự cố.

Tổng quan kỹ thuật #

Web tĩnh SPA: React + TypeScript + Vite + Tailwind CSS + shadcn/ui, state bằng Zustand. Backend tuỳ chọn: Supabase (Postgres + Auth + Realtime). Không cần server riêng.

  • Chạy 2 chế độ: localStorage (không .env) hoặc online (có .env Supabase).
  • Đa phòng khám (multi-tenant): 1 project Supabase phục vụ nhiều phòng khám, phân biệt bằng slug.

Cài đặt & chạy #

npm install
npm run dev        # phát triển (http://localhost:5173)
npm run build      # xuất thư mục dist/
npm run preview    # xem thử bản build

Kiến trúc & luồng dữ liệu #

Draft / Published — store src/store/content.ts:

  • content = bản nháp (admin sửa); published = bản website hiển thị (site đọc s.published).
  • set() sửa nháp + bật dirty; save() chép nháp → published + đóng dấu pubTs; discard() huỷ nháp.

Đồng bộ (src/lib/sync.ts): đẩy Supabase khi published đổi (đã Lưu), kèm updated_at = pubTs. Khi nhận dữ liệu (fetch đầu & realtime) chỉ áp khi bản server mới hơn (so updated_at) và luôn giữ nguyên bản nháp đang sửa (chống “tự quay về như cũ”). Bố cục & Áp dụng mẫu gọi save() ngay.

Khung Xem trước ở trang Bố cục là một iframe chạy chính app (cùng origin, dùng chung localStorage). merge() của store chặn không cho ngữ cảnh khác ghi đè bản nháp chưa lưu.

Biến môi trường (.env) #

VITE_SUPABASE_URL=https://xxxx.supabase.co
VITE_SUPABASE_ANON_KEY=sb_publishable_...   # hoặc eyJhbGci...
VITE_SITE_SLUG=tuyetlan                      # slug phòng khám cho bản deploy này

Bỏ trống 3 dòng → chế độ localStorage. VITE_SITE_SLUG phải khớp một dòng companies.slug trong DB.

Supabase & schema #

  1. Tạo project tại supabase.com (free).
  2. Chạy schema: SQL Editor → dán toàn bộ supabase/schema.sql → Run. Tạo bảng companies, memberships, site_content, bookings + RLS + hàm setup_clinic. Chạy 1 lần.
  3. Lấy khoá: Project Settings → API → Project URL + anon/publishable key → điền .env.

Đăng nhập & tự cấu hình #

  1. Tạo user admin: Authentication → Users → Add user (email + mật khẩu).
  2. Đăng nhập /admin lần đầu: app tự chạy ensureTenant — tạo công ty theo VITE_SITE_SLUG (nếu chưa có) và gắn tài khoản vào công ty. Không cần chạy SQL tay.

Làm thủ công (hoặc hàng loạt) vẫn được:

select public.setup_clinic('tuyetlan', 'Phòng khám Tim mạch Tuyết Lan', 'email-admin@example.com');

Nhiều admin / phòng khám #

  • Thêm admin cùng phòng khám: tạo user mới ở Authentication → họ đăng nhập /admin là tự gắn vào công ty của slug đó.
  • Mở phòng khám mới: bản deploy đặt VITE_SITE_SLUG=slug-moi + tạo user → đăng nhập lần đầu tự tạo công ty mới. Dữ liệu tách theo company_id.

Bảo mật (Row Level Security) #

  • bookings: khách (anon) chỉ được gửi; chỉ thành viên công ty mới đọc/sửa/xoá.
  • site_content: ai cũng đọc (hiển thị website); chỉ thành viên mới ghi.
  • companies: đọc công khai (phân giải theo slug); tạo bởi user đã đăng nhập.
Admin đăng nhập nhưng chưa là thành viên công ty → danh sách đặt lịch trống. Trang Đặt lịch sẽ hiện cảnh báo kèm lệnh setup_clinic.

Email & xuất báo cáo #

  • Email đặt lịch: dán Web3Forms Access Key trong Sao lưu & Email (không cần server). Có sẵn hook gọi Edge Function send-booking-email nếu triển khai.
  • Excel: SheetJS (xlsx) tạo .xlsx thật (cột + auto-filter), nạp động khi bấm.
  • PDF: mở cửa sổ báo cáo & dùng chức năng in của trình duyệt — in đúng màu/định dạng như trên màn hình, tự đóng cửa sổ khi in xong/huỷ.

Triển khai (Vercel) #

  1. Đẩy mã nguồn lên GitHub.
  2. vercel.com → Add New Project → chọn repo (tự nhận Vite).
  3. Environment Variables: VITE_SUPABASE_URL, VITE_SUPABASE_ANON_KEY, VITE_SITE_SLUG.
  4. Deploy. vercel.json đã rewrite mọi path về index.html để /admin hoạt động khi F5.

Reset dữ liệu #

Xoá trình duyệt: Sao lưu & Email → “Xoá dữ liệu trình duyệt” (xoá các khoá tl_* trong localStorage).

Xoá sạch database (giữ bảng/RLS/hàm), rồi đăng nhập lại để tự dựng lại:

truncate table public.companies cascade;

Xử lý sự cố #

Đặt lịch không hiện trong admin

  • Tài khoản chưa gắn công ty → đăng xuất/đăng nhập lại (tự gắn), hoặc chạy setup_clinic('slug','tên','email').
  • Kiểm tra Console (F12) dòng [backend].

Website trống / sai nội dung

Kiểm tra VITE_SITE_SLUG khớp companies.slug đã tạo.

Sửa nội dung không thấy đổi

Nhớ bấm Lưu thay đổi; nếu online, tài khoản phải là thành viên công ty để ghi được.
